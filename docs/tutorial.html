限频规则 · 5 分钟入门

让 ClassIsland 自动化"别太勤快"——限定一段时间内最多跑几次。三种模式 / 三张示例 / 七个问答。

plugin classisland.plugin.ratelimit version 1.0.0 audience 学生 / 教师 read 5 min status draft

§1这是什么

装上这个插件后，你可以在 ClassIsland 的自动化里加一种叫 「限频规则」 的东西。它做的事情非常直白：

❯ 限频：告诉 ClassIsland，这个自动化最多跑 N 次
⏱ 限频：时间间隔 A
◉ 限频：时间点 B
▣ 限频：时间段 C
↑↓ navigate ↵ select esc close

比如你的"上课响铃"自动化，触发器设得敏感一点的话，一节课可能响 五六次——这就尴尬了。装上限频规则后，你可以告诉它：

「一节课，最多响 两次。超过就不响，下次再说。」

就这样。规则自己会数，自动在合适的时候"闭嘴"。

§2我能用它做什么

下面是三个最常遇到的场景。每个场景告诉你用哪个模式。具体字段含义见 §4。

限频：时间间隔 01

上课铃别太勤

"上课事件触发后，45 分钟内最多响 1 次。"

限频：时间点 02

整点提醒吃饭

"8:00 / 12:00 / 18:00 整点提醒，1 小时内最多 1 次。"

限频：时间段 03

晚自习播报

"19:00 ~ 21:30 内最多播报 5 次。"

§3怎么装

  1. 打开 GitHub Releases，下载最新的 ClassIsland.RateLimit.cipx 文件。
  2. 打开 ClassIsland 主界面 → 设置 → 插件 → 安装本地插件。
  3. 选择刚才下载的 .cipx 文件，重启 ClassIsland（完全退出再打开）。
  4. 打开任一自动化的 规则集 编辑器，右键 → 添加规则，搜"限频"应能看到三个。
提示 · tip

如果你不熟悉 ClassIsland 的规则集怎么用，先去 ClassIsland 官方文档了解下基础概念（什么是规则 / 规则组 / 行动）。

§4三种模式怎么选

三种模式各有各的"专长"。先看总表：

name
trigger
fit for
A
时间间隔Interval
距上次执行 ≥ 触发间隔，且窗口内次数 < 上限，才放行。
事件触发频繁、又怕重复响的场景。
B
时间点Time Point
当前时间在配置的某个 HH:mm ± 1 分钟内，才放行。
整点打卡、定时播报。
C
时间段Time Range
当前时间在 [开始, 结束] 区间内（支持跨夜），才放行。
晚自习、上班、熄灯前等场景。

模式 A · 时间间隔 —— 详细字段

每条规则都有两个公共字段 + 一个专属字段。先看公共：

● common.cs2 properties
最大次数3窗口内最多跑几次
时间窗口600统计最近多久的历史（秒）

模式 A 多了这一项：

● IntervalRateLimitSettings.cs1 property
触发间隔60距上次执行 ≥ 这个值才放行（秒）

模式 B · 时间点 —— 详细字段

● TimePointRateLimitSettings.cs1 property
时间点08:00,12:00,18:00每个时刻的 ± 1 分钟内都算"命中"

模式 C · 时间段 —— 详细字段

● TimeRangeRateLimitSettings.cs2 properties
开始时间19:00从这一刻起开始放行
结束时间21:30到这一刻为止放行
小贴士 · cross-midnight

想"22:00 到次日 06:00"生效？直接开始=22:00、结束=06:00，插件会自动识别为跨夜区间。

§5举个完整的栗子

假设你想做一个"晚自习提醒喝水"的自动化：

目标 · goal

晚自习 19:00 ~ 21:30 内，每 30 分钟最多提醒喝水 1 次。

步骤

  1. 新建自动化，触发器选 定时器 / 心跳（每 1 分钟检查一次）。
  2. 进入规则集，添加规则 「限频：时间段」，按上面的字段填好。
  3. 添加行动：显示提醒（内容写"该喝水啦 💧"）。
  4. 保存，启用自动化。

效果

✅ 19:00 提醒一次，30 分钟内不重复（即使心跳每分钟跳）。
✅ 19:30 再次提醒一次。
✅ 21:30 之后进入"非时间段"，安静收工。

注意 · caveat

限频规则只在它放行的时候记一次。如果你的行动失败/取消了，这条记录不会回滚。
如果你希望"只有行动真执行了才算一次"，可以在末尾加一个 「记录限频执行」 行动，并关掉规则的自动记录。

§6常见问题

装上后怎么没看到规则？›

先确认 ClassIsland 已完全重启（不是最小化、是真的退出再打开）。然后去任一自动化的规则集编辑器里 右键 → 添加规则，搜"限频"应能看到三个。

为什么我的规则不生效？›

常见原因：

① 这条规则放在了"或"分支下面 —— 改成放在"与"分支最稳。

② 你的工作流没有挂这个规则（规则只对它所属的工作流生效）。

③ 时间窗口设得太短，调大到 60 秒以上再试。

④ 查看 ClassIsland 日志，搜"限频"会告诉你为什么没放行。

窗口里的历史能清空吗？›

可以。插件本身不提供"清空"按钮，但通用做法是关掉自动化 → 等几分钟（让历史自然过期）→ 再开。窗口设得越大，等得越久。

「记录限频执行」行动是干嘛的？›

普通模式：规则放行 → 立即记一次。

「记录」模式：只有行动真执行完了才记一次。

后者更精确（避免"规则过了但行动失败了"还占用名额），但配置稍麻烦。一般 用普通模式就够了。

会不会影响电脑性能？›

完全不会。插件只在 ClassIsland 里跑，历史数据按工作流 GUID 存到本地 JSON，每秒最多被读/写几十次，占用可忽略。

三个模式可以同时用吗？›

可以。在同一规则组的"与"分支里同时加"时间间隔"和"时间段"，那么"晚自习每 30 分钟"这种组合就出来了。

怎么反馈问题？›

去 GitHub 仓库的 Issues 页面提，记得附上 ClassIsland 版本、插件版本和日志。

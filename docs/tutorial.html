1. 这是什么？

装上这个插件后，你可以在自动化里加一种叫 "限频规则" 的东西。

它的作用很简单：

🛑 告诉 ClassIsland：「这个自动化别跑太勤，限定时间内最多跑几次」

比如上课铃自动化每节课响一次，本来挺好的。但万一你的触发器很敏感，一节课它响 5、6 次，就尴尬了。

加了限频规则后，你可以告诉它："1 节课最多响 2 次"。超了就不响了，下次条件触发再说。

2. 我能用它做什么？

🔔 上课铃别太勤

"上课事件已经触发了，45 分钟内最多响 1 次"

用 时间间隔 模式，触发间隔 45 分钟。

⏰ 整点提醒

"8:00 / 12:00 / 18:00 整点提醒吃饭，1 小时内最多 1 次"

用 时间点 模式，时间点 08:00,12:00,18:00。

📚 晚自习播报

"19:00 - 21:30 内最多播报 5 次"

用 时间段 模式，开始 19:00、结束 21:30。

🪟 下课自动开窗

"下课事件触发后，5 分钟内最多开 1 次窗"

用 时间间隔 模式，触发间隔 5 分钟。

3. 怎么装？

  1. 打开 GitHub Releases 页面： https://github.com/ywydog/RateLimit/releases 下载最新的 ClassIsland.RateLimit.cipx 文件。
  2. 打开 ClassIsland 主界面 → 设置 → 插件 → 安装本地插件。
  3. 选择刚才下载的 .cipx 文件，重启 ClassIsland。
  4. 打开任一自动化的 规则集 编辑，右键 → 添加规则 → 你能看到三种带"限频"字样的规则。
💡 小贴士：如果你不熟悉 ClassIsland 的规则集怎么用，先去 ClassIsland 官方文档了解下基础概念会更顺。

4. 三种规则怎么选？

三种模式各有各的"专长"，先用一张表对比一下：

模式 A

⏱️时间间隔

每隔一段时间最多跑一次。

适合：事件触发很频繁、又怕重复响的场景。

模式 B

📍时间点

只在固定时刻（08:00、12:00…）生效。

适合：整点打卡、定时播报。

模式 C

🌙时间段

每天某个时段内才生效（支持跨夜）。

适合：晚自习、上班时间、熄灯前。

模式 A · 时间间隔 —— 详细配置

每条规则都有两个公共字段 + 一个专属字段：

最大次数3 次（窗口内最多跑几次）
时间窗口600 秒（统计最近 10 分钟）
触发间隔60 秒（距上次执行 ≥ 60s 才放行）

例子：想让"下课响铃" 5 分钟内最多 1 次，就这样填：

最大次数1
时间窗口300 秒
触发间隔300 秒

→ 效果：每次下课事件触发，先看过去 5 分钟有没有响过；有就跳过、没有才响。

模式 B · 时间点 —— 详细配置

把要触发的时刻写进一个列表，多个用英文逗号分隔：

最大次数2 次（同一时刻段最多响 2 次）
时间窗口60 秒
时间点08:00,12:00,18:00

→ 效果：只在 8:00、12:00、18:00 这 3 个时刻的 ± 1 分钟内放行；其他时间一律不响。

📝 时间格式：必须是 HH:mm，比如 08:00、21:30。不写秒。

模式 C · 时间段 —— 详细配置

直接用时间选择器选开始/结束：

最大次数5 次（整个晚自习最多 5 次）
时间窗口9000 秒（覆盖 2.5 小时）
开始时间19:00
结束时间21:30

→ 效果：只在 19:00 ~ 21:30 之间放行；最多 5 次。

🌙 跨夜写法：想让"熄灯提醒"在 22:00 到次日 06:00 生效？ 直接填 开始 22:00、结束 06:00，插件会自动识别为跨夜区间。

5. 举个完整的栗子 🌰

假设你想做一个"晚自习提醒喝水"的自动化：

目标

晚自习 19:00 ~ 21:30 内，每 30 分钟最多提醒喝水 1 次。

步骤

  1. 新建自动化，触发器选"定时器 / 心跳"（每 1 分钟检查一次）。
  2. 进入规则集，添加规则：限频：时间段。
    最大次数1
    时间窗口1800 秒 (30 分钟)
    开始时间19:00
    结束时间21:30
  3. 行动：显示提醒（"该喝水啦 💧"）。
  4. 保存，启用自动化。

效果

✅ 19:00 提醒一次，30 分钟内不重复（即使心触发每分钟跳一次）。

✅ 19:30 再次提醒一次，然后 30 分钟内不再响。

✅ 21:30 之后进入"非时间段"，安静收工。

⚠️ 重要：限频规则只在它"放行"的时候记一次。
如果你的自动化失败/取消了，这条记录不会回滚。 如果你希望"只有行动真执行了才算一次"，可以在末尾加一个"记录限频执行"行动，并关掉规则的自动记录。

6. 常见问题

我装上后怎么没看到规则？

先确认 ClassIsland 已完全重启（不是最小化、是真的退出再打开）。然后去任一自动化的规则集编辑器里右键 → 添加规则，搜"限频"应该能看到三个。

为什么我的规则不生效？

常见原因：
① 这条规则放在了规则的"或"分支下面 —— 改成放在"与"分支最稳。
② 你的工作流没有挂这个规则（规则只对它所属的工作流生效）。
③ 时间窗口设得太短，调大到 60 秒以上再试。
④ 查看 ClassIsland 日志，搜"限频"会告诉你为什么没放行。

窗口里的历史能清空吗？

可以的。插件本身不提供"清空"按钮，但 ClassIsland 通用做法是关掉自动化 → 等几分钟（让历史自然过期） → 再开。窗口设得越大，等得越久。

"记录限频执行"行动是干嘛的？

普通模式：规则放行 → 立即记一次。
"记录"模式：只有行动真执行完了才记一次。
后者更精确（避免"规则过了但行动失败了"还占用名额），但配置稍麻烦。一般用普通模式就够了。

会不会影响我电脑性能？

完全不会。插件只在 ClassIsland 里跑，历史数据按工作流 GUID 存到本地 JSON，每秒最多被读/写几十次，占用可忽略。

三个模式可以同时用吗？

可以。在同一规则组的"与"分支里同时加"时间间隔"和"时间段"，那么"晚自习每 30 分钟"这种组合就出来了。

怎么反馈问题？

去 GitHub 仓库的 Issues 页面提，记得附上 ClassIsland 版本、插件版本和日志。
